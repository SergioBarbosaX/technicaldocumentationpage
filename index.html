Introduction

AngularJS is a JavaScript-based open-source front-end web framework mainly maintained by Google and by a community of individuals and corporations to address many of the challenges encountered in developing single-page applications. It aims to simplify both the development and the testing of such applications by providing a framework for client-side model–view–controller (MVC) and model–view–viewmodel (MVVM) architectures, along with components commonly used in rich Internet applications. (This flexibility has led to the acronym MVW, which stands for "model-view-whatever" and may also encompass model–view–presenter and model–view–adapter.) In 2014, the original AngularJS team began working on the Angular web framework.

The AngularJS framework works by first reading the Hypertext Markup Language (HTML) page, which has additional custom HTML attributes embedded into it. Angular interprets those attributes as directives to bind input or output parts of the page to a model that is represented by standard JavaScript variables. The values of those JavaScript variables can be manually set within the code, or retrieved from static or dynamic JSON resources.

AngularJS is the frontend part of the MEAN stack, consisting of MongoDB database, Express.js web application server framework, Angular.js itself, and Node.js server runtime environment.


Setup

AngularJS is a JavaScript framework. It is a library written in JavaScript.

AngularJS is distributed as a JavaScript file. To add AngularJS to a web page we should set a script tag with a src value like this:

                
                    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.7.7/angular.min.js">

Once this line is added to our html file we can begin to use the AngularJS framework.

It is recommended to load the AngularJS library either in the <head> or at the start of the <body>. This is because calls to Angular modules (some kind of modules that we will see later) can only be compiled after the library has been loaded.


Directives

The way that AngularJS works is extending html tags with directives. Every directive has a ng- prefix, so this is the reason that this tags are called ng-directives.

There are different kinds of directives that we can use, but the most importants to begin to code are:

  • The ng-app directive defines an AngularJS application.
  • The ng-model directive binds the value of HTML controls (input, select, textarea) to application data.
  • The ng-bind directive binds application data to the HTML view.

Lets take a look to our first example.

                
                    <!DOCTYPE html>
                    <html>
                    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.7.7/angular.min.js">
                    <body>
                    <div ng-app="">
                      <p>Name: <input type="text" ng-model="name"></p>
                      <p>ng-bind="name"</p>
                    </div>
                    </body>
                    </html>

The explanation of the above code is:

  1. AngularJS starts automatically when the web page has loaded due to the script tag.
  2. The ng-app directive tells AngularJS that the div element is the "owner" of an AngularJS application.
  3. The ng-model directive binds the value of the input field to the application variable name.
  4. The ng-bind directive binds the content of the p element to the application variable name.
  • The ng-init directive initializes application data.

We can see this directive in an example.

                
                    <div ng-app="" ng-init="firstName='John'">
                    <p>Name: <input type="text" ng-model="firstName"></p>
                    <p>You wrote: {{ firstName }}</p>
                    </div>

The code above shows in the screen the name John after the sentence You wrote: because John is the value that we have initialized and assigned to the variable {{firstName}}

The {{firstName}} expression, in the example above, is an AngularJS data binding expression.

Data binding in AngularJS binds AngularJS expressions with AngularJS data.

{{firstName}} is bound with ng-model="firstName".

  • The ng-repeat directive repeats an HTML element.
                
                    <div ng-app="" ng-init="names=['Jani','Hege','Kai']">
                    <ul>
                        <li ng-repeat="x in names">
                          {{ x }}
                        <li>
                    </ul>
                    </div>

The code above shows in the screen an unordered list with the names Jani, Hege, Kai


Expressions

AngularJS expressions can be written inside double braces: {{ expression }}.

AngularJS expressions can also be written inside a directive: ng-bind="expression".

AngularJS will resolve the expression, and return the result exactly where the expression is written.

AngularJS expressions can contain literals, operators, and variables.

We can use diferent kind of elements in an AngularJS expression.

  • AngularJS numbers
                
                    <div ng-app="" ng-init="quantity=1;cost=5">
                    <p>Total in dollar: {{ quantity * cost }}</p>
                    </div>

The same code using ng-bind

                
                    <div ng-app="" ng-init="quantity=1;cost=5">>
                    <p>Total in dollar: <span ng-bind="quantity * cost"></span>
                    </div>

Where the result is 5 in both cases.

  • AngularJS strings
                
                    <div ng-app="" ng-init="firstName='John';lastName='Doe'">
                    <p>The name is {{ firstName + " " + lastName }}</p>
                    </div>

Where the result is: The full name is John Doe

The same example can be done using ng-bind directive.

  • AngularJS objects
                
                    <div ng-app="" ng-init="person={firstName:'John',lastName:'Doe'}">
                    <p>The name is {{ person.lastName }}</p>
                    </div>

Where the result is: The name is Doe

The same example can be done using ng-bind directive.

  • AngularJS arrays
                
                    <div ng-app="" ng-init="points=[1,15,19,2,40]">
                    <p>The third result is {{ points[2] }}</p>
                    </div>

Where the result is: The third result is 19

The same example can be done using ng-bind directive.


Controllers

AngularJS controllers control the data of AngularJS applications.

AngularJS applications are controlled by controllers.

The ng-controller directive defines the application controller.

                
                    <div ng-app="myApp" ng-controller="myCtrl">
                    First Name: <input type="text" ng-model="firstName"><br>
                    Last Name: <input type="text" ng-model="lastName"><br>
                    <br>
                    Full Name: {{firstName + " " + lastName}}

                    </div>

                    <script>
                    var app = angular.module('myApp', []);
                    app.controller('myCtrl', function($scope) {
                    $scope.firstName = "John";
                    $scope.lastName = "Doe";
                    });
                    </script>

The application works in this way:

  1. The AngularJS application is defined by ng-app="myApp". The application runs inside the </div>.
  2. The ng-controller="myCtrl" attribute is an AngularJS directive. It defines a controller.
  3. The myCtrl function is a JavaScript function.
  4. AngularJS will invoke the controller with a $scope object.
  5. In AngularJS, $scope is the application object (the owner of application variables and functions).
  6. The controller creates two properties (variables) in the scope (firstName and lastName).
  7. The ng-model directives bind the input fields to the controller properties (firstName and lastName).

The result of the application is Full name: John Doe

A controller is a JavaScript Object, created by a standard JavaScript object constructor.

One more example

In this case the object has attributes and methods.

                
                    <div ng-app="myApp" ng-controller="personCtrl">
                    First Name: <input type="text" ng-model="firstName"><br>
                    Last Name: <input type="text" ng-model="lastName"><br>
                    <br>
                    Full Name: {{fullName()}}

                    </div>

                    <script>
                    var app = angular.module('myApp', []);
                    app.controller('personCtrl', function($scope) {
                        $scope.firstName = "John";
                        $scope.lastName = "Doe";
                        $scope.fullName = function() {
                            return $scope.firstName + " " + $scope.lastName;
                        };
                    });
                    </script>

The result of the application is like the previous example Full name: John Doe

In larger applications, it is common to store controllers in external files. To do this we only need to store the code between <script></script> in a file with .js extension and call it from our file in this way <script src="personController.js"></script>


Modules

An AngularJS module defines an application.

The module is a container for the different parts of an application.

The module is a container for the application controllers.

Controllers always belong to a module.

To create a module we need to use the AngularJS function angular.module

                
                    <div ng-app="myApp">...</div>
                    <script>
                        var app = angular.module("myApp", []);
                    </script>

The "myApp" parameter is an HTML element in which the application will run.

The [] parameter in the module definition can be used to define dependent modules.

Without the [] parameter, we are not creating a new module, but retrieving an existing one.

Once the module is created we can add a controller with the ng-controller directive or other kind of things to our AngularJS application.

                
                    <div ng-app="myApp" ng-controller="myCtrl">
                    {{ firstName + " " + lastName }}
                    </div>
                    <script>
                    var app = angular.module("myApp", []);

                    app.controller("myCtrl", function($scope) {
                    $scope.firstName = "John";
                    $scope.lastName = "Doe";
                    });
                    </script>

It is common in AngularJS applications to put the module and the controllers in JavaScript files.

                
                    <!DOCTYPE html>
                    <html>
                    <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.7.7/angular.min.js">
                    <body>
                    <div ng-app="myApp" ng-controller="myCtrl">
                    {{ firstName + " " + lastName }}
                    </div>
                    <script src="myApp.js"></script>
                    <script src="myCtrl.js"></script>
                    </body>
                    </html>

"myApp.js" contains an application module definition, while "myCtrl.js" contains the controller.


Model

The ng-model directive bind the value of an input field to a variable created in AngularJS.

                
                    <div ng-app="myApp" ng-controller="myCtrl">
                        Name: <input ng-model="name">
                        <h1>You entered: {{name}}</h1>
                    </div>
                    <script>
                    var app = angular.module("myApp", []);
                    app.controller("myCtrl", function($scope) {
                        $scope.name = "John Doe";
                    });
                    </script>

In the previous example when data in the input field is updated, the value displayed is updated too.

The ng-model directive can provide type validation for application data (number, e-mail, required)

                
                    <form ng-app="" name="myForm">
                    Email:
                    <input type="email" name="myAddress" ng-model="text">
                    <span ng-show="myForm.myAddress.$error.email">Not a valid e-mail address>
                    </form>

In this case the span will be displayed only if the expression in the ng-show attribute returns true.

The ng-model directive can provide status for application data (valid, dirty, touched, error)

                
                    <form ng-app="" name="myForm" ng-init="myText = 'post@myweb.com'">
                    Email:
                    <input type="email" name="myAddress" ng-model="myText" required>
                    <h1>Status</h1>
                    {{myForm.myAddress.$valid}}
                    {{myForm.myAddress.$dirty}}
                    {{myForm.myAddress.$touched}}
                    <form>

The ng-model directive provides CSS classes for HTML elements, depending on their status.

Some of those classes are

  • ng-empty
  • ng-not-empty
  • ng-touched
  • ng-untouched
  • ng-valid
  • ng-invalid
  • ng-dirty
  • ng-pending
  • ng-pristine
                
                    <style>
                    input.ng-invalid {
                        background-color: lightblue;
                    }
                    </style>
                    <body>
                    <form ng-app="" name="myForm">
                    Enter your name:
                    <input name="myName" ng-model="myText" required>
                    </form>

Data Binding

An AngularJS application follows the MVC pattern.

Data Binding refers to the synchronization between the model and the view. That means that if model changes view reflects it and vice-versa if view changes model is updated as well.

The data model is a collection of data available for the application.

                
                    var app = angular.module("myApp", []);

                    app.controller("myCtrl", function($scope) {
                    $scope.firstName = "John";
                    $scope.lastName = "Doe";
                    });

Now from the view we can access to the data model. We can do this in different ways:

  • ng-bind directive
    •                     
                        <p ng-bind="firstname"></p>
  • double braces {{ }}
    •                     
                        <p>First name: {{firstname}}</p>
  • ng-model directive
    •                     
                        <input ng-model="firstname">
